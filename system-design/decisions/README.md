# Decisions

Architecture Decision Records (ADRs) documenting key choices made during development.

## Format

Filename: `YYYY-MM-DD-title.md`

```markdown
---
date: YYYY-MM-DD
type: decision
status: Accepted | Superseded | Deprecated
author: <see authorship convention in CLAUDE.md>
---

# Decision: [Title]

## Context

What's the situation? What forces are at play?

## Decision

What did we decide?

## Consequences

What becomes easier or harder? What are the tradeoffs?

## Alternatives Considered

What else did we think about? Why didn't we choose those?
```

## Authorship

All files follow the authorship convention defined in CLAUDE.md. Use the `RAW-` prefix only for 100% human-authored content. Include `author:` in frontmatter metadata.
